Passo 1: Criando o Projeto Vue.js com Vite

Navegue até o diretório que você criou:


cd seudiretório

Crie o projeto Vue.js usando Vite: Execute o comando para iniciar o projeto com o Vite:


npm create vite
  • Nome do Projeto: Será solicitado um nome para o projeto. Você pode nomeá-lo conforme preferir, por exemplo: gerenciamento-salas.
  • Selecione a Framework: Selecione Vue como o framework.
  • Selecione a variante: Escolha Vue (sem TypeScript por enquanto, mas você pode optar por Vue + TypeScript se quiser usar TS).
  • Instale as dependências: Após a criação do projeto, entre na pasta do projeto e instale as dependências:

cd gerenciamento-salas
                

npm install

Execute o projeto:


npm run dev

Isso deve abrir a aplicação Vue.js padrão no navegador.

Passo 2: Organizando a Estrutura de Pastas

Com o projeto criado, vamos organizar as pastas para seu sistema de gerenciamento de salas. Aqui está uma sugestão para organizar as páginas e componentes:

  • Estrutura Inicial de Diretórios: Abaixo está um esboço de como você pode estruturar o projeto de acordo com as funcionalidades mencionadas.

│   │   └── common/

│   │       ├── Sidebar.vue
│   ├── pages/
│   │   ├── Home.vue
│   │   ├── Login.vue

  • Explicação das Pastas:
    • components/: Contém componentes reutilizáveis. Por exemplo, o Navbar.vue para o menu de navegação, o Sidebar.vue para a barra lateral e componentes de formulário (como login e registro).
    • pages/: Contém as páginas principais do projeto, cada uma representando uma funcionalidade (login, registro, visualização e adição de salas e laboratórios).
    • router/: Contém a configuração de rotas para navegar entre as páginas do sistema.
    • store/: Contém o estado global da aplicação (se usar Vuex ou Pinia para gerenciamento de estado).

Passo 3: Configurando o Router

Para gerenciar as rotas entre as páginas (como login, home, etc.), precisamos configurar o vue-router.

  • Instalar o vue-router: No diretório do projeto, rode:

npm install vue-router
  • Configurar o router: Crie o arquivo src/router/index.js com a seguinte configuração básica:

import { createRouter, createWebHistory } from 'vue-router';
import Home from '../pages/Home.vue';
import Login from '../pages/Login.vue';
import Register from '../pages/Register.vue';
import VisualizarSala from '../pages/VisualizarSala.vue';
import VisualizarLaboratorio from '../pages/VisualizarLaboratorio.vue';
import AdicionarSala from '../pages/AdicionarSala.vue';
import AdicionarLaboratorio from '../pages/AdicionarLaboratorio.vue';
import AdicionarRecurso from '../pages/AdicionarRecurso.vue';

const routes = [
  { path: '/', name: 'Home', component: Home },
  { path: '/login', name: 'Login', component: Login },
  { path: '/register', name: 'Register', component: Register },
  { path: '/visualizar-sala', name: 'VisualizarSala', component: VisualizarSala },
  { path: '/visualizar-laboratorio', name: 'VisualizarLaboratorio', component: VisualizarLaboratorio },
  { path: '/adicionar-sala', name: 'AdicionarSala', component: AdicionarSala },
  { path: '/adicionar-laboratorio', name: 'AdicionarLaboratorio', component: AdicionarLaboratorio },
  { path: '/adicionar-recurso', name: 'AdicionarRecurso', component: AdicionarRecurso },
];

const router = createRouter({
  history: createWebHistory(),
  routes,
});

export default router;
  • Atualizar o main.js para usar o router: No arquivo src/main.js, importe o router e configure a aplicação para usá-lo:

import { createApp } from 'vue';
import App from './App.vue';
import router from './router';

createApp(App)
  .use(router)
  .mount('#app');

  • Exemplo de Componente de Formulário de Login (LoginForm.vue): Crie o arquivo src/components/form/LoginForm.vue.

Passo 5: Executar e Testar

npm run dev

Arquivos e Trechos para Remover ou Modificar

Remover o conteúdo de exemplo em App.vue: O arquivo App.vue geralmente contém código de exemplo e estilos que não são relevantes para o seu projeto. Você pode limpar o conteúdo e manter apenas uma estrutura básica.

Arquivo: src/App.vue

Remover o componente de exemplo HelloWorld.vue: Por padrão, o Vite cria um componente chamado HelloWorld.vue como exemplo. Ele pode ser removido, já que não será necessário.

Arquivo: src/components/HelloWorld.vue

  • Ação: Exclua o arquivo, pois ele é apenas um exemplo.

Remover o conteúdo da pasta assets/: A pasta assets/ contém imagens e arquivos de exemplo que o Vite coloca por padrão (como o logotipo do Vue). Se você não for utilizar, pode remover esses arquivos.

Arquivos: Imagens e recursos na pasta src/assets/.

  • Ação: Exclua ou substitua esses arquivos conforme necessário.

Remover importações desnecessárias em main.js: O arquivo main.js pode conter algumas importações de estilos ou de componentes que foram removidos.

Arquivo: src/main.js

  • Modificação: Certifique-se de que ele esteja limpo e importe apenas o necessário, como o Vue e o Router.

Explicação: Essa configuração mínima permite que a aplicação inicie com o roteador (para navegação entre páginas) e o componente principal App.vue.

Excluir estilos padrão no App.vue: O arquivo App.vue pode incluir estilos padrão que não são necessários, como estilos globais de exemplo. Limpe esses estilos para começar com uma base mais limpa.

Arquivo: src/App.vue

  • Ação: Exclua quaisquer estilos desnecessários, especialmente os que vêm com exemplos.

Ajustar o título no index.html: O arquivo index.html gerado pelo Vite contém um título padrão ("Vite App"). Você pode alterar isso para refletir o nome do seu projeto.

Arquivo: index.html

  • Explicação: Atualizar o título melhora a identificação da aplicação quando for acessada no navegador.

Arquivos Opcionalmente Removíveis

  • Remover arquivos de configuração desnecessários (opcional): Dependendo das ferramentas que você utilizará, pode ser que alguns arquivos de configuração não sejam necessários para o seu projeto. Por exemplo:
    • vite.svg: Se você não for usar a logo do Vite ou imagens SVG.
    • public/vite.svg: Logo padrão do Vite que pode ser removida.
  • Ajustar ou remover diretivas de ESLint/Prettier (opcional): Caso o projeto tenha configurações de linting (como ESLint ou Prettier) geradas automaticamente, verifique se os padrões estão de acordo com o que você prefere. Caso contrário, ajuste ou remova as configurações.
    • Arquivos: .eslintrc.js, .prettierrc
    • Ação: Ajuste ou remova conforme suas preferências de estilo de código.

Recapitulando

Remover arquivos de exemplo: HelloWorld.vue e imagens da pasta assets/.

Limpar o App.vue: Manter uma estrutura mínima.

Ajustar o main.js: Garantir que ele só tenha as importações necessárias.

Modificar o título no index.html: Altere para refletir o nome do projeto.

Verificar e remover configurações de linting ou ferramentas não utilizadas.

Com esses ajustes, seu projeto Vue.js estará mais limpo e pronto para iniciar o desenvolvimento das funcionalidades que você descreveu.

Crie um Arquivo de Estilos Globais

Crie um arquivo CSS para estilos globais:

Na pasta src/assets/styles/, crie um arquivo chamado global.css. Você pode ajustar o caminho conforme sua estrutura de diretórios.

Estrutura do diretório:


src/
├── assets/
│   └── styles/
│       └── global.css
├── components/
├── views/
└── main.js

Adicione estilos ao global.css:


/* src/assets/styles/global.css */

/* Estilos globais */
body {
  font-family: Arial, sans-serif;
  margin: 0;
  padding: 0;
  background-color: #f7f7f7;
}

h1, h2, h3, h4, h5, h6 {
  margin: 0;
  padding: 0;
}

a {
  color: #007bff;
  text-decoration: none;
}

a:hover {
  text-decoration: underline;
}

/* Adicione mais estilos globais conforme necessário */

Importe o Arquivo de Estilos no main.js

Abra o arquivo main.js localizado em src/main.js.

Importe o arquivo global.css no início do arquivo main.js:


// src/main.js

import { createApp } from 'vue';
import App from './App.vue';
import router from './router';

// Importa os estilos globais
import './assets/styles/global.css';

const app = createApp(App);

app.use(router);

app.mount('#app');

Resultado

Agora, qualquer estilo definido no global.css será aplicado a toda a aplicação. Por exemplo, a fonte padrão e as cores de link definidos no global.css aparecerão em todos os componentes da aplicação, a menos que sobrescritos por estilos específicos dentro dos componentes.

Conclusão

Arquivo global.css: Contém estilos que devem ser aplicados a toda a aplicação.

Importação em main.js: Garante que os estilos globais sejam carregados quando a aplicação é inicializada.

Seguindo esses passos, você mantém a aplicação organizada e garante que os estilos globais sejam aplicados consistentemente em toda a aplicação Vue.js.

Certifique-se de que o axios está instalado

Verifique se o pacote axios está instalado no seu projeto. Caso não esteja, você pode instalá-lo usando o seguinte comando:


npm install axios

Verifique a Importação

Certifique-se de que você está importando o axios corretamente no seu componente. A importação deve ser:


import axios from 'axios';

Verifique a Configuração do vite.config.js

Se você estiver usando o Vite, pode haver um problema com a configuração do Vite. Verifique se a configuração do seu vite.config.js está correta e se não há problemas com a resolução de módulos.

Verifique se o axios está listado em package.json

Abra seu package.json e verifique se o axios está listado na seção de dependências:


"dependencies": {
  "axios": "^x.x.x", // Verifique se a versão está listada
  ...
}
